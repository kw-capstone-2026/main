# XGBoost 베이스라인 모델 상세 기술 명세 (Technical Details)

본 문서는 `final_merged_commercial_data.parquet`을 활용한 베이스라인 모델이 **AUC 0.90**이라는 높은 성능을 기록한 이유와 적용된 주요 하이퍼파라미터 설정을 상세히 설명합니다.

## 1. 모델 아키텍처 및 하이퍼파라미터 설정

XGBoost 모델의 성능과 안정성을 위해 다음과 같은 파라미터를 최적화하여 적용했습니다.

| 하이퍼파라미터         | 설정값  | 기술적 의도 및 효과                                                                                                                                 |
| :--------------------- | :------ | :-------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`n_estimators`**     | `200`   | 너무 적으면 과소적합, 너무 많으면 과적합될 수 있으나, 조기 종료 없이 안정적인 학습을 위해 200회로 설정.                                             |
| **`max_depth`**        | `5`     | 트리의 깊이를 5로 제한하여 모델이 너무 복잡해지는 것을 방지하고 일반화 성능을 높임.                                                                 |
| **`learning_rate`**    | `0.05`  | 학습 속도를 낮게 유지하여 손실 함수(Loss Function)의 최솟값에 더 정교하게 도달하도록 유도.                                                          |
| **`scale_pos_weight`** | `33.33` | **[가장 중요]** 정상 상권 대비 위험 상권의 비율(약 33:1)을 가중치로 주어, 데이터가 적은 '위험 상권'을 더 적극적으로 찾도록 함. (Recall 향상의 핵심) |
| **`reg_lambda`**       | `10`    | L2 규제(Regularization)를 강하게 주어 피처 간의 영향력이 특정 변수에만 쏠려 과적합되는 것을 방지.                                                   |
| **`random_state`**     | `42`    | 결과의 재현성을 위해 난수를 고정하여 누가 돌려도 동일한 결과를 보장.                                                                                |

## 2. 높은 성능(AUC 0.90)의 핵심 요인 분석

### ① 강력한 선행 지표 (Leading Indicators)

단순한 매출뿐만 아니라 **`폐업_영업_개월_평균`**과 **`폐업_률`** 같은 과거의 '실패 기록' 데이터가 포함된 것이 결정적이었습니다.

- 망한 가게들이 버틴 기간이 짧아질수록 해당 상권의 위험도가 급격히 올라가는 패턴을 모델이 매우 명확하게 잡아냈습니다.

### ② 데이터 정제 및 타입 일치

문자열로 섞여 있던 수치들을 `Float64`로 정확히 변환하여, 정보 손실 없이 모든 수치적 관계를 XGBoost가 계산할 수 있도록 환경을 조성했습니다.

### ③ 시계열적 타당성 (Time-series Integrity)

데이터를 무작위로 섞지 않고 2019~2023년의 흐름을 학습시킨 뒤 2024년을 예측하게 함으로써, 모델이 상권의 '성장-정체-쇠퇴' 흐름을 논리적으로 이해하게 되었습니다.

## 3. 팀원들을 위한 가이드

팀원들이 새로운 보조 데이터(예: 학교 거리, 인구 밀도 등)를 추가할 때, 이 베이스라인 모델 설정값을 유지한 채 변수만 추가하여 비교하는 것이 가장 정확합니다.

- 만약 새 데이터를 넣었을 때 **AUC가 0.90보다 올라간다면**, 그 데이터는 과포화 예측에 실질적인 기여를 하는 '유의미한 피처'라고 결론지을 수 있습니다.
- 반대로 AUC가 변하지 않거나 떨어진다면, 해당 데이터는 기존 변수들과 중복되거나 노이즈일 가능성이 높습니다.
