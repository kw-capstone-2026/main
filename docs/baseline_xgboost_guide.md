## 데이터 병합(Join) 시 필수 주의사항

팀원들이 각자 가져온 보조 데이터를 마스터 테이블에 합칠 때, 아래 5가지 원칙을 반드시 지켜야 데이터 왜곡을 방지할 수 있습니다.

### 1. 조인 키(Join Keys) 타입 일치

- **원칙:** 모든 데이터의 `기준_년분기_코드`와 `상권_코드`는 **정수형(`int64`)**이어야 합니다.
- **조치:** 병합 전 `df['상권_코드'] = df['상권_코드'].astype('int64')` 처리를 생활화하세요.

### 2. 1:1 대응 확인 (Aggregation)

- **원칙:** 마스터 테이블은 [분기+상권]당 1행입니다.
- **조치:** 추가 데이터가 업종별/격자별로 여러 행을 가지고 있다면, 반드시 `groupby(['기준_년분기_코드', '상권_코드']).mean()` 등으로 **집계 후 병합**하세요. 그렇지 않으면 데이터 행 수가 폭발적으로 늘어납니다.

### 3. Left Join 사용

- **원칙:** 베이스라인의 512,626행 규모를 유지해야 합니다.
- **조치:** 항상 베이스라인을 왼쪽에 두고 `pd.merge(base, extra, how='left', on=...)` 방식을 사용하세요.

### 4. 필요한 컬럼만 로드

- **원칙:** 대용량 파일(예: `df_final`) 전체를 메모리에 올리는 것은 비효율적입니다.
- **조치:** `pd.read_parquet(path, columns=['필요한_컬럼_리스트'])`를 사용하여 필요한 피처만 선택적으로 로드하세요.

### 5. 결측치(NaN) 전략 수립

- **원칙:** 병합 후 발생하는 `NaN`은 데이터 부재를 의미합니다.
- **조치:** 해당 피처의 특성에 따라 `0`으로 채울지, `평균값`으로 보간할지 결정하세요. (XGBoost 모델은 `NaN`을 그대로 처리할 수도 있지만, 분석 의도에 따라 처리가 필요할 수 있습니다.)
