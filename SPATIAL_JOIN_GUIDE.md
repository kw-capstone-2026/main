---

## 🎯 목표

**서울시 생활인구 데이터를 상권 분석에 활용 가능한 형태로 가공**

- 목적: 소상공인 창업 상권 과포화 예측 플랫폼 개발
- 핵심 과제: 격자 단위 원본 데이터 → 상권 단위 시계열 데이터 변환

## 🔄 데이터 처리 과정

### Phase 1: 시간 차원 압축 (일 → 분기)

**목표:** 일별 시간별 데이터를 분기별 시나리오로 집계

**방법론:**
```python
1. 데이터 정제
   - 마스킹 처리 ("*" → 0.0)
   - 인코딩 변환 (CP949 → UTF-8)
   - 주중/주말 분류

2. 시나리오 설계 (6개)
   - 주중: 오전(7-9시), 점심(11-14시), 저녁(18-21시), 야간(21-23시)
   - 주말: 주간(10-18시), 야간(18-23시)
   
3. 분기별 집계
   - 월별 → 분기별 평균
   - 2023Q1 ~ 2026Q2 (14분기)
```

**결과:**
```
grid_quarterly_all.parquet
- 8,893개 격자 × 14분기 = 120,546행
- 파일 크기: 25MB
```

---

### Phase 2: 공간 차원 압축 (격자 → 상권)

**핵심 과제:** 격자 좌표 정보 부재 문제 해결

#### 문제점
```
서울시 원본 데이터:
✅ grid_id: "다사35005075"
✅ 인구 통계
❌ 위도, 경도 (제공하지 않음!)

→ Spatial Join 불가능
```

#### 해결 과정

**1단계: 패턴 분석**
```python
grid_id 구조: "다사" + 8자리 숫자
예: "다사35005075"
  → 3500 (X 격자 번호)
  → 5075 (Y 격자 번호)
```

**2단계: 통계청 100m 격자 데이터로 검증**
```python
100m 격자 TM 좌표 분석:
- X: 900,050 ~ 999,950
- Y: 1,900,050 ~ 1,999,950

서울 범위:
- X: 940,000 ~ 970,000
- Y: 1,930,000 ~ 1,960,000

→ 추정 공식 검증 완료
```

**3단계: 좌표 변환 공식 도출**
```python
TM 좌표 = 기준점 + (격자번호 × 25m)

기준점:
- X: 852,500
- Y: 1,838,125

변환:
TM_X = 852,500 + (3500 × 25) = 940,000
TM_Y = 1,838,125 + (5075 × 25) = 1,965,000

좌표계 변환:
TM (EPSG:5179) → WGS84 (EPSG:4326)
```

**4단계: Spatial Join**
```python
geopandas.sjoin(
    격자_point,
    상권_polygon,
    predicate='within'
)

결과:
- 매칭된 격자: 216개 (상권 내 격자만)
- 상권 수: 210개
- 미매칭 격자: 8,677개 (주거지, 산, 공원 등)
```

**5단계: 상권별 집계**
```python
groupby(['상권_코드', '년분기']).agg({
    '시나리오별_인구': 'sum',
    '격자_수': 'count'
})
```

## 🔬 질적 지표 계산

### 1. 주야간 비율
```python
주야간비율 = (주간인구 평균) / (야간인구 평균)

활용: 직장 상권 vs 주거 상권 구분
- 비율 > 1: 직장 밀집 (낮 유동인구 많음)
- 비율 < 1: 주거 밀집 (밤 유동인구 많음)
```

### 2. 성장 트렌드 (population_slope)
```python
선형 회귀로 분기별 인구 변화 기울기 계산

활용: 상권 성장/쇠퇴 판단
- 양수: 유동인구 증가 추세
- 음수: 유동인구 감소 추세
```

### 3. 인구 집중도 (population_entropy)
```python
엔트로피 = -Σ(p_i × log(p_i))
(상권 내 격자별 인구 분포)

활용: 상권 활성화 패턴
- 높음: 골고루 분포 (면적 상권)
- 낮음: 특정 지점 집중 (핫플레이스)
```
